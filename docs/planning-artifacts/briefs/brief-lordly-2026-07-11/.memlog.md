- (decision) Stack: Phaser 3 + TypeScript web game (PWA, mobile-first Android); project lives at ~/projects/js/lordly — chosen for link-share play and no app store friction
- (decision) MVP is battle-only: no world map, no unit movement, no strongholds — the battle IS the game; whoever wins the battle wins, GG
- (decision) Each player picks any 3 of 6 classes, duplicates allowed (balance stats, not rules, if degenerate combos appear)
- (decision) Classes: Knight (front tank, 2 front attacks), Archer (back row, 2 shots, reaches back row), Mage (row AoE, beats knights), Cleric (heals, weak attack), Witch (OB64-style status effects, no damage), Zombie (neutral placeholder, average stats, no strengths/weaknesses)
- (decision) RPS triangle: mage beats knight, knight beats archer, archer beats mage; cleric/witch support; zombie neutral
- (decision) Placement: hidden simultaneous (Battleship-style) on own 3x3 formation grid; OB64 formation feel — rows matter; max 5 units mid-late game, 3 in MVP
- (decision) Battle-end: BOTH modes wanted — single engagement (each unit spends attack count once, remaining HP judges) AND until-wipeout (repeating engagements); wipeout may defer to later stage
- (decision) Combat presentation: animated OB64-style auto-battle scene (indirect control is the heart of the game feel)
- (decision) Opponents: vs AI first (whole game playable solo), 1v1 via shareable link (Node + WebSocket) as its own later epic
- (change) Scope cut during brainstorm: round-1 answer was turn-based tactical grid with moving squads; user overrode in round 2 — board movement/world map removed from MVP entirely
- (assumption) Post-MVP direction: 5-unit squads, more classes, possibly strategic map later; zombie is a placeholder name pending user decision
- (event) Installer left literal {output_folder} dirs (unresolved variable); removed empty placeholders, using docs/planning-artifacts instead
- (event) Brainstorm captured from ogre_game.txt + 3 rounds of Q&A; discovery largely complete before brief drafting
- (decision) Name: display title 'Lord Battle Tactics' (double homage: Ogre Battle + Tactics Ogre); repo/slug/handle stays 'lordly'
- (decision) Stakes: passion project — me + friends, fun first; success = full match on phone is fun + friend clicks link and asks rematch; public release is maybe-later
- (decision) Art: free/CC fantasy pixel-art packs (itch.io, OpenGameArt, Kenney) for MVP — retro OB64-adjacent feel, zero art production
- (decision) Witch kit: sleep/stun, poison, weaken, and confusion (confused unit may attack an ally or heal an enemy); per-engagement casting rules deferred to PRD
- (event) Landscape research digest received: exact niche (hidden formation + 1v1 + browser + challenge link) unoccupied; Mechabellum closest cousin (visible boards, PC-only); OB64 nostalgia demand active; auto-battler churn drivers = shop RNG, solved meta, monetization
